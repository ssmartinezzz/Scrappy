# GGA Code Review Rules — Fashion Scraper

Lean, high-signal review checklist used by the pre-commit hook. Keep this file
SHORT: its full text is prepended to every review prompt, so length directly
drives review latency. It replaces the full `CLAUDE.md` as `RULES_FILE`. Review
the diff for merge-blocking defects only — not style preferences.

## Verdict contract
- Respond with exactly one line: `STATUS: PASSED` or `STATUS: FAILED`.
- FAIL only on a concrete, merge-blocking defect. When unsure, PASS.
- Do not flag pre-existing code outside the changed lines.

## Block on (STATUS: FAILED)
1. **Correctness**: logic that produces wrong results, off-by-one, inverted
   conditions, null/empty/`Optional` mishandling, unhandled exceptions on a
   reachable path.
2. **Concurrency & DB**: writes to `DatabaseService` outside the `writeLock`;
   reads that reuse the write `Connection` instead of `readConn`/`readLock`;
   SQL built by string concatenation of untrusted input (use parameterized
   statements); transactions left uncommitted / not rolled back on error.
3. **Contract breaks**: changing a REST response shape, DB column, or
   `Product`/`MlScore` record field without updating all producers/consumers;
   badge keys must be the canonical set (`all_time_low`, `below_market`,
   `verified_deal`, `trending`, `price_dropping`, `above_market`,
   `fake_discount`) — no old Spanish snake_case literals in production code.
4. **Resource leaks**: unclosed `Connection`/`Statement`/`ResultSet`/Playwright
   pages/streams; missing try-with-resources.
5. **Security**: hardcoded secrets/tokens; path traversal; unsanitized input
   reaching a shell/subprocess (`PythonRunner`), SQL, or file path.
6. **Python ML**: bare `except:` that swallows a real failure silently on a new
   code path; scoring that uses gondola price where unit price is required
   (packs must score on `precio/cantidadUnidades`); mutation of shared holders
   at import time.
7. **Frontend**: `twMerge`/`cn()` toggling `display` classes (strips `flex` —
   use inline style); Tailwind `/opacity` modifiers on `var()` tokens (compiles
   to nothing); stale-closure state in effects/pagination; keys not stable.
8. **Regressions**: removing or weakening an assertion, guard, or test without
   justification.

## Do NOT block on
- Formatting, naming, comment wording, import order.
- Spanish vs English identifiers (project mixes both intentionally).
- Missing tests when the change is a trivial/mechanical edit.
- Subjective architecture preferences that are not defects.

## Project quick facts (only what a reviewer needs)
- Java 21 + Spring Boot; Python 3.11 ML subprocess; React 18 + Vite frontend.
- SQLite single-writer: all writes serialized under `writeLock`; dedicated
  `readConn` under `readLock` for standalone reads (WAL).
- ML scoring uses unit price for packs; display/discount/history use gondola price.
